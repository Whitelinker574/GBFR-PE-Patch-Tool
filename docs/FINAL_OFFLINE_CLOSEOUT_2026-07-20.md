# GBFR Data Tools 离线收尾与公式采样交付报告（2026-07-20）

## 结论

当前机器不连接游戏进程也能完成的代码、数据、存档副本事务、UI、原生应用回归、公式采样器和说明文档已收尾。发布候选程序为 `build\bin\GBFRDataTools-final.exe`。

本轮没有把“无游戏进程无法验收”写成成功：运行时写入/注入、真实最终面板回读与公式证据采集仍需等游戏可用后执行，操作方法已给出。

## 本轮完成内容

### 配装与存档写入

- 1269 项真实因子背包改为容器自适应虚拟网格，最多 6 列、固定行高与越界一行预渲染；4K 也不超过 60 张 DOM 卡片，可滚到、选中第 1269 项，搜索后滚动位置正确复位。
- 保留“构造模式”为默认；“从背包选择”仍是上方 12 槽、下方搜索与因子列表。主/副词条、真实因子图标与替换槽互动保留。
- 专精位于因子下方，默认折叠但三个方向常驻可见。展开后有 1/2/3/EX 四阶与真谛/觉醒/秘义三组；1 阶允许三方向技能，2 阶起非主方向具名专精技能锁定，子词条仍按游戏规则处理。
- 配装、已验证的武器第七阶效果和 4 个召唤石实例可在同一次离线事务写入。一次读取/预检、共享写入锁、校验和、原子替换与写后重开回读，任一 stale guard 不匹配就整批拒绝。
- 离线生成器统一共享存档变更锁。失败重试只能重试同一输出路径与同一次已写字节；源或输出被外部修改就 fail-closed，成功加载新存档会清理旧失败状态。

### 数值、武器技能与召唤石

- 真实存档固定基准已拆分角色基础、Fate/命运篇章、Master/角色永久强化、武器基础值和角色强化伤害上限。伊欧真存档回归的无装备固定基准为 `9796 HP / 3831 ATK / 5% Crit / 8 Stun / +100% DmgCap`。
- 武器技能显示存档真实等级、解锁阶段、条件和效果；右侧合并同类加成但保留来源账本，普通/能力/奥义伤害上限独立展示。
- 离线最终 HP/攻击/暴击/昏厥/伤害上限仍显示 `≈`和“草稿公式未完全验证”，不把尚未闭环的百分比乘区、条件效果与中间 `float32` 量化冒充游戏精确面板。
- 召唤石实体 `189/189`、天然主加护 `82/82`、可计算副参数均已由本地 2.0.2 表联结。三份 Excel 分别是副参数、基础/突破哈希和 189 个实体 ID，不是“缺 60 项主加护定义”。

### 只读公式监控程序

- 新页面：“内存监测（只读） → 角色公式采样”。只用 `PROCESS_QUERY_INFORMATION | PROCESS_VM_READ` 打开独立句柄，不复用编辑器句柄，不安装 Hook/代码洞，不写进程、存档或网络。
- 强制一轮只选一个实验类型：空白对照、武器、武器技能、因子、专精、上限突破、召唤石、HP 条件、战斗条件或其他；按 `A1 → B1 → A2 → B2` 采集，并强制 `A2 == A1` 与 `B2 == B1`。
- 每个阶段在 24 KiB 状态窗口采样前后各读取 3 次最终面板，状态窗口自身也连续读取 3 次。面板或状态不稳定、角色/版本/进程身份变化都拒绝当轮。
- 脱敏包格式为 `.gbfr-formula-sample.zip`，内含 `manifest.json`、`events.json`、`observations.json`、`candidates.json`、`formula-model.json`、`redaction-report.json`、`README.txt`与 `SHA256SUMS`。不含原始内存、PID、模块基址、绝对地址、本地路径、用户名、绝对时间或自由备注。
- 采集与导出时都使用 `VirtualQueryEx` 删除实际映射指针，并对四阶段指针掩码取并集。候选 JSON 只留状态对象内相对偏移、i32/f32 类型和 A→B 差值；查询失败按 fail-closed 处理。

## 等游戏可用后的操作

### 第一批：先收集公式证据

1. 运行 `build\bin\GBFRDataTools-final.exe`，进入“内存监测（只读） → 角色公式采样”。
2. 对每个目标角色先做一轮“空白对照”：A1/B1/A2/B2 全程什么都不改。
3. 再按因子 → 召唤石 → 专精 → 武器 → 武器技能 → 可逆上限突破 → HP 条件 → 战斗条件的顺序分别采样；每个包只改一项。
4. 每轮导出 `.gbfr-formula-sample.zip`，不要解压、不要修改，直接交给我分析。
5. 每个包附上：角色名、B 相对 A 唯一改变的项目/等级/数值、城镇/训练场/任务地点、HP/战斗条件以及是否为重复实验。

详细点击顺序、命名和失败条件见 [`角色公式采样操作说明.md`](角色公式采样操作说明.md)。Master/Fate/角色等级等不可逆永久成长不应强行放进同一进程的 A/B/A/B；它们需要以后准备成对存档或匹配角色协议。

### 第二批：再做运行时 E2E

公式只读采样结束后，再独立验收因子、祝福、召唤石、上限突破、货币/药水、素材/掉落、怪物增强和 CT 生产目录的连接、启用、冲突、停止、断开与退出恢复。这些结果只能由实际 2.0.2 进程验收，不能用离线单元测试代替。
