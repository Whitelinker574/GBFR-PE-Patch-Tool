# 离线功能收口与公式采样器设计

## 目标

在不连接 GBFR 2.0.2 进程的前提下，完成当前能够安全实现和自动验证的产品缺口，并交付一个供后续实机阶段使用的严格只读公式采样器。采样器由用户按 A1/B1/A2/B2 向导操作，导出本地脱敏包，再由开发侧分析公式、候选偏移和重复性。

本设计不宣称离线人物公式已经精确，也不把静态 EXE、合成内存测试或三快照一致性误写成真实进程验收。

## 范围拆分

### 一、配装编辑离线收口

1. 背包因子使用固定高度网格虚拟化。1269 项真实存档下，页面仍显示总数，但 DOM 中只保留可见行和两行 overscan；滚动到底、搜索归零、点击替换槽位均保持现有语义。
2. 配装编辑页新增武器实例展开区。第一版只允许编辑已被 DLC 2.0.2 目录和现有后端证明安全的第七阶段四选一效果，不暴露任意 Hash，也不伪装 `2818[0:4]`、三组武器词条或 `2814/2816` 已经可安全改写。
3. 配装编辑页新增四颗召唤石实例展开区，可编辑天然主加护、副参数、对应等级和 Rank。未知旧主加护只能原样保留；不能替换成未验证值。
4. 武器和召唤石草稿按稳定 SlotID 建立，不原地修改加载上下文。仅 dirty 草稿进入提交。
5. 所有配装、因子构造、武器和召唤石修改必须在一次 `LoadoutApplyRequest` 中完成：一次加载、全量预检、一次内存缓冲修改、一次校验和、一次原子落盘、一次重新打开回读。
6. 第一版编辑的是背包中的全局物品实例。界面明确提示同一实例被其他配装引用时会同步受影响；不实现尚无权威空槽构造语义的武器/召唤石复制。

### 二、离线 UI 与数据收口

1. 角色长名使用单行省略和 `title`，头像、名字、配装数量保持横向。
2. 即时因子、离线因子和表单使用内容宽度上限，避免 1059px/1159px 的无意义拉伸。
3. 普通页面立绘改为严格右下锚定；独立配装编辑页继续隐藏立绘。
4. 修正六个召唤主加护可见名称，统一为“躲避仇火”“钳蟹的报恩”“属性克制转换”和“α/β/γ秘纹”。
5. 图标只从用户提供的游戏资源、解包表和参考库建立可证明映射。不能证明的条目继续使用中性回退，不凭名称猜图。
6. UI 保持旧版无角色花纹羊皮纸、暖色标签和左面板/右立绘产品结构。设计检索中的霓虹复古方案与既有产品规范冲突，不采用。

### 三、公式采样器

采样器作为“内存监测”分类下的独立页面，不复用 `CT084RuntimeMonitor`、`CharaAcquire` 或选中物品 Hook。

#### 只读进程边界

Windows Adapter 只允许：

- `PROCESS_QUERY_INFORMATION`
- `PROCESS_VM_READ`
- `VirtualQueryEx`
- `ReadProcessMemory`

采样器源码和依赖中禁止：

- `PROCESS_ALL_ACCESS`
- `PROCESS_VM_WRITE`
- `PROCESS_VM_OPERATION`
- `WriteProcessMemory`
- `VirtualProtectEx`
- `VirtualAllocEx`
- `CreateRemoteThread`
- 网络上传客户端

页面销毁、进程身份变化、版本守卫失败、采样取消和应用退出都必须关闭只读句柄。

#### 采样流程

每个实验固定为 A1/B1/A2/B2：

1. A1：记录原始配置的三次 bit-exact 稳定快照。
2. B1：用户在游戏中只改一个变量后记录。
3. A2：恢复原配置后记录，用于证明可逆。
4. B2：再次应用同一变化，用于证明可重复。

变化类型限定为 weapon、weapon_skill、sigil、mastery、overlimit、summon、hp_condition、battle_condition、control、other。自由备注默认不导出。

已知探针记录最终 HP、攻击、昏厥、暴击，以及当前静态 EXE 证据支持的基础、Master、Fate 子字段。每个数值同时记录展示值和原始 bit pattern，证据等级分开表示，不再由单个 `FormulaVerified` 布尔值概括。

高级模式允许只读扫描角色状态对象 `status+0x0000..0x6000` 的对齐 i32/f32 单元，并用 A/B/A/B 的可逆性和 control 事件收敛候选。全进程扫描不作为默认流程；若提供，只读取已提交且非 guard/noaccess 的私有可读页，设置扫描字节数、耗时和候选数量硬上限，不保留原始内存块。

#### 导出格式

导出扩展名为 `.gbfr-formula-sample.zip`，仅通过本地保存对话框生成。包内包含 manifest、events、observations、candidates、formula-model、redaction-report、README 和 SHA256SUMS。

默认移除 PID、绝对创建时间、模块基址、堆地址、存档/EXE 绝对路径、用户名、角色自定义名称、自由备注、原始内存块和本地绝对时间。已知状态对象只保留相对偏移，堆候选使用匿名编号。

## 数据流与事务

配装写入路径：

`前端草稿 -> LoadoutApplyRequest -> 全量快照校验 -> prepared mutations -> 单缓冲写入 -> FixChecksums -> 原子保存/备份 -> 重新读取 -> 逐字段验证`

采样路径：

`实验向导 -> 只读短句柄 -> 版本/身份守卫 -> 三快照一致性 -> 本地会话状态 -> 脱敏序列化 -> ZIP + SHA256SUMS`

两条路径不共享进程句柄。保存编辑器在游戏运行时仍拒绝写存档；采样器永远不写存档或游戏内存。

## 错误处理

- 任一配装资源快照过期、引用歧义、目录不合法或字段缺失，都必须在落盘前失败。
- 同一请求对同一实例给出不同修改时拒绝；相同修改去重。
- 回读不一致时返回明确的备份恢复错误，不把部分结果标记为成功。
- 采样中进程重启、PID 复用、角色 Hash 变化、版本守卫变化或三快照不一致时废弃当前事件。
- 扫描超预算应失败关闭，而不是截断后冒充完整结果。

## 验收

1. 1269 项真实存档下，背包虚拟区 DOM 卡片不超过 60，末项可滚动、搜索和选择。
2. 武器合法四选一及召唤石合法主/副词条可在配装事务内写入副本并回读；未提交实例逐字节不变。
3. 省略 inline edit 时保持当前 `1451` 只改引用且 `1456..1460` 不变的兼容行为。
4. 新采样模块通过源码安全扫描，无法链接任何写内存或联网能力。
5. 假内存测试覆盖稳定快照、进程身份、版本守卫、A/B/A/B 收敛、取消、预算、脱敏和 ZIP 校验和。
6. Node、Go、Vite、Wails、go vet、真实存档副本回读、配装原生交互和多尺寸页面矩阵全部重新运行。
