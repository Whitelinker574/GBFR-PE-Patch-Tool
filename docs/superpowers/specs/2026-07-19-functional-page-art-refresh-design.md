# 功能页立绘与表情包去重设计

日期：2026-07-19

## 目标

严格使用 `GBFR-Relink-Portrait-Skill-Portable-20260718-v3.zip` 的页面分析、page brief、官方身份参考、两阶段立绘、Q 版贴纸和校验流程，为新增 CT 页面制作功能匹配资产，并消除当前两个页面映射复用与一张被裁切立绘。

## 当前审计

- 当前 17 个工具页映射 15 套视觉资产；生产文件没有同字节副本。
- `loadout` 与 `loadoutPresets` 共用芙劳立绘和贴纸。
- `wrightstone` 与 `wrightstoneMemory` 共用菲莉立绘和贴纸。
- `loadout-official-edge-safe.webp` 虽命名为 edge-safe，实际主体高度约 99.2%，顶部仅 0.8% 空隙且底部贴边，未通过 Portable 校验。
- 实时配装页当前芙劳贴纸与工作流 accepted example 完全一致，语义匹配，应保留。
- 新增 `ctCombat`、`ctCharacters`、`ctQuest` 后若继续复用旧图，会放大用户指出的重复与功能不协调问题。

## 页面 brief 与资产清单

每个页面先写 JSON brief 并运行 `validate_page_brief.py`。生产目标为 **6 张新立绘、5 张新贴纸，共 11 张资产**：

1. `loadout` 实时录制/复刻
   - 角色：芙劳；保留身份和现有贴纸。
   - 动作：按 12 槽顺序核对并移动一个真实因子记录标记，双手关系清楚。
   - 目的：重做完整 edge-safe 立绘，修复底部裁切，不改变页面角色语气。
2. `loadoutPresets` 离线配装预设
   - 立绘：格兰；展开一份配装卷册，明确整理武器、12 因子、4 技能与专精，不复制实时录制动作。
   - 贴纸：格兰；认真确认备份和目标角色，单一检查手势，无文字气泡。
3. `wrightstoneMemory` 实时祝福三槽
   - 立绘：玛琪拉菲菈；在一个三槽祝福石校准架上装入最后一枚词条石，官方浮游剑保持完整且不变形成道具。
   - 贴纸：玛琪拉菲菈；写入后提醒重新选中目标，拿单颗三槽祝福石并做清楚的“再选一次”手势，无文字。
4. `ctCombat` 战斗规则
   - 立绘：巴恩；在训练场用盾与单一校准装置测试格挡/闪避规则，动作是测试而非攻击敌人。
   - 贴纸：巴恩；明确“地址不符就停手”的停止手势，盾牌数量和手臂结构正确。
5. `ctCharacters` 角色机制
   - 立绘：姬塔；以团长身份检查角色机制卡组和队伍轮换，卡片数量有限且排列可读。
   - 贴纸：姬塔；认真提醒冲突机制不能同时开启，用单一互斥拨杆表达。
6. `ctQuest` 任务与便利
   - 立绘：尤达拉哈；对照任务地图标记隐藏宝箱、倒计时和支线目标，只有一张地图与三个明确标记。
   - 贴纸：尤达拉哈；提醒返回默认状态再离开任务，用收起任务地图的动作表达。

角色选择以 Portable `references/roster.json` 的 29 个 slug 为准；官方身份图仅来自 `assets/official/<slug>.png`。现有 roster 外兼容页资产不作为新图身份参考。

## 生成流程

1. 读取真实 Vue 组件、API、验证、禁用态、成功态和最新截图。
2. 为六页分别保存 `tools/portrait-jobs/briefs/<page>.json`，填写页面目标、工作流、核心动作、风险和 UI 气泡文案。
3. 按精确可见名称/ID 从本地 2.0.2 semantic catalog 检索至多三张物品/因子/技能参考；图标优先直接用于 UI，不让模型重画小图标。
4. 校验 brief 后运行 `prepare_portrait_job.py`；先生成并审查 2048×2048 线稿，再生成 2160×3840 高质量立绘。
5. 除实时配装页外运行 `prepare_sticker_job.py`，生成 2048×2048 贴纸；对白只放 HTML/CSS 气泡，不烘焙到图片。
6. 每张透明成品以原分辨率检查身份、手指、武器连续性、道具数量、边缘和透明角，并运行 `validate_portrait.py`。
7. 只把通过的成品转为生产 WebP；PNG 原图、brief、manifest 和元数据保留在被 `.gitignore` 排除的工作目录，不提交 API 密钥或压缩包。

## 集成与裁切

- 生产路径继续使用 `frontend/src/assets/gbfr/cutouts/` 与 `stickers/`。
- 新页文件名为 `ct-combat.webp`、`ct-characters.webp`、`ct-quest.webp`；现有页保持语义文件名。
- `PatchTool.vue` 一页一映射；`loadout`、`loadoutPresets`、`wrightstone`、`wrightstoneMemory` 不再共享映射。
- 添加映射完整性测试：每个非首页工具页都必须显式拥有立绘和贴纸；生产代码不得用 `progressionArt`/`progressionSticker` 静默兜底新增页面。
- CSS 为个别角色提供 position/scale token，禁止通过全局缩小所有立绘掩盖一页裁切。
- 默认窗口允许在大腿附近裁切，但头、发、双手、功能道具和武器关键部位必须完整；右栏保持现有大幅立绘比例，不放回小卡片容器。

## 视觉 QA

- 在 375、768、1024、1440 宽度和最低支持高度下逐页截应用内部画面。
- 同时检查完整右栏、折叠右栏和全屏；人物不能覆盖左栏文字/按钮，气泡与贴纸不重叠。
- 贴纸额外在约 150×160 显示尺寸检查情绪和手势。
- 生成资产做感知哈希和映射审计：生产文件不得同字节重复，新三页不得复用旧映射。
- 若任一图身份漂移、动作与页面无关、道具结构不明、手脚异常或碰边，则只修一个缺陷后重新验证。
