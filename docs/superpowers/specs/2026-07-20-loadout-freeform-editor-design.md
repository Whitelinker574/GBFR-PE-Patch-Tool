# 配装自由编辑与专精推导设计

## 目标

配装编辑器必须优先表达用户实际写入意图：因子由因子种类、主词条、副词条和三组等级独立组成；专精方向由已点节点推导；自然合法性只提示，不阻止可编码的存档写入。同时修复窄栏武器技能编辑的溢出。

## 交互设计

- 因子构造器只搜索因子目录，不把背包现有实例伪装成构造模板。
- 主词条与副词条均使用完整特性目录，可搜索、可清空副词条，名称不附加“强制组合”等状态后缀。
- 当前组合偏离自然生成目录时，在合规区域显示警告与原因；保存按钮保持可用。
- 专精区不提供方向选择按钮。每次增删节点后，方向按二阶各分类的节点数自动推导；未达门槛或混合配置时显示说明，但不删节点、不禁用其他方向节点。
- 三方向卡片直接显示已选专精技能的真实效果；详细节点仍可展开检查。
- 武器技能编辑行在侧栏内始终采用单列布局，文字和下拉框都允许收缩和换行。

## 写入边界

继续阻止无法安全编码的结构错误：不存在的目录 ID、未知 hash、重复实际槽位、数组越界、资源容量不足、错误角色的专精节点、单阶节点超过存档容量。以下情况降级为警告并允许写入：非自然主副词条组合、超自然等级但仍在可写范围、混合专精方向、未点满专精。

## 验证

以单元测试覆盖自由主副词条、警告不阻塞、自动方向推导、专精效果可见和窄栏单列布局；再用真实测试存档启动 Wails 应用，在多种窗口尺寸下完成构造、保存、回读与截图检查。
