## 结论先行

本轮已将离线存档、运行时注入、运行时监测和工具设置分成四个顶层分类。对本地 **CT 0.8.4** 的可枚举直接补丁候选，安全覆盖为 **60/64**：58 个进入新的生产目录，2 个复用项目已有的更安全实现；另 4 个因上游自述不安全或在锁定的 2.0.2 EXE 中签名多命中而明确拒绝。没有为凑数而暴露无法证明安全的开关。

当前可以做出以下结论：

- **CT 直接补丁中能安全落地的部分无已知遗漏。** 58 个新目录功能分为战斗 9、角色 37、任务/便利 12；连同 2 个已有实现，覆盖为 60/64。
- **其余 57 个叶脚本不能被粗暴记作 57 个“未集成功能”。** 它们是代码洞、Lua、参数或游戏内保存动作，其中部分已有结果等价实现，其余需要真实进程、寄存、场景和生命周期验证；未验证项不提供伪按钮。
- **CT 0.8.4 没有离线 `SaveData.dat` 编辑。** 项目的离线存档页是独立解析与写入实现；即使结果可等价，也不应记作“移植了 CT 的离线功能”。
- **游戏最终人物面板已有精确只读回读路径。** 读取器从 2.0.2 游戏状态对象获取 HP、攻击、昏厥和暴击；只有版本守卫通过且连续 3 份快照一致时才标记 `runtimeVerified`。
- **离线人物数值仍是显式 `≈` 估算。** Fate、Master 和基础四项已根据真实存档/解包表修正，但部分百分比 producer、条件效果和中间 `float32` 量化顺序尚未完全闭环，不会冒充游戏最终值。
- **最终发布门禁尚待最新立绘/图标入库后再跑一次。** 全量测试数、最终 EXE/DLL 大小与哈希、22 个顶层页面状态的截图矩阵均以本报告 TODO 为准，未预填旧数字。

因此，不能把“缺少完整游戏进程验收”、“尚未逆完的公式/资源”或“最终产物未固化”写成已经完成。

### CT 与离线存档的边界

CT 0.8.4 **没有离线 `SaveData.dat` 解析或文件写入**。它的“仓库/保存”项通过 `openProcess`、NBLib、代码洞和游戏内保存调用作用于正在运行的进程。本项目的离线因子、祝福、配装、专精、召唤石等页面是独立的存档格式实现；即使可以产生部分结果等价的数据，也不应记作“移植了 CT 的离线存档功能”。

## 本轮主要实现

### 1. 配装编辑器

- 编辑配装改为进入独立大页面；正常配装列表仍保留右侧角色立绘，编辑态隐藏立绘以腾出空间。
- 默认进入“构造因子”模式；上方固定展示 12 个槽位、当前因子和主/副词条，下方才是搜索、构造或背包候选。
- 背包候选使用独立卡片，构造模式以替换槽为核心；写入时才生成新因子并绑定配装。
- 支持真实武器、武器技能、4 个可装备主动技能槽、4 个召唤石、12 个因子和完整专精配置。
- 单个角色的一套配装支持 JSON 导入/导出；保留槽位顺序并报告缺失实例，不做用户已取消的批量导出。
- 写入采用计划、备份、写入、校验和回滚流程；新因子、技能、专精、召唤石与角色记录同步写入。

### 2. 专精规则与界面

- 专精放在因子配置下方并可完整展开，结果汇总到右侧。
- 展示三条专精方向及 R1、R2、R3、EX 阶段。
- R1 的三方向根技能可以并存；从 R2 起只能形成一个主方向，R3 必须沿用该方向。
- 没有选择主方向时，R2/R3 普通子节点允许保留，但不会伪造对应方向的根效果。
- 等级配额、阶段依赖、方向冲突和导入数据由前后端共同校验。

### 3. 属性结算与真实面板

- 头像下方显示 HP、攻击力、暴击率、昏厥值和伤害上限五项结果。
- 因子、武器技能、专精、角色上限突破和召唤石进入汇总；同类加成先聚合，再按等级曲线封顶。
- 伤害上限保留普通、能力、奥义三类拆分，不强行合并成一个虚假值。
- 右侧来源账本明确默认计算范围；未知召唤效果显示哈希和“未计入”，不静默消失。
- 精确只读路径从 RVA `0x7C24980` 的角色状态 manager 经 ID vector 与 hash map 定位目标角色，读取状态对象 `+0x04/+0x08/+0x10/+0x14` 的 HP、攻击、昏厥、暴击。
- 回读受 RVA `0xD4321`、`0x2DC081`、`0x2DC11E`、`0xA296F3` 四处 2.0.2 字节守卫保护，并要求连续 3 份快照完全一致才返回 `runtimeVerified=true`；该路径句柄只有查询与 `PROCESS_VM_READ` 权限。
- 真实伊欧存档 `1309/1310/1312/1313` 分别给出 `3156 HP / 666 ATK / 8 Stun / 5 Crit`；`1318=0x7FF` 联 Fate 表得 `HP +640 / ATK +165`，`1323=3,309,499` 联 Master 表并钳制 Lv50 得 `HP +6000 / ATK +3000 / DmgCap +100`，无装备基础线为 `9796 HP / 3831 ATK / 8 Stun / 5 Crit`。
- 离线四项继续显示 `≈`；只有真实游戏回读结果显示精确值。公式证据、RVA、字段、舍入点和真实回归值见 [`FORMULAS_2.0.2.md`](FORMULAS_2.0.2.md)。

### 4. 因子、祝福和数据真值

- `GEEN_156_24` 按本地 2.0.2 `gem.tbl` 修正为主词条 `SKILL_023_00 / 0xCAC6AFF2`，不再误用 `SKILL_156_00`。
- 91 个角色专属因子按本地 `gem.tbl` 和二词条 lot 联表：25 项固定副词条，66 项使用已验证随机池；固定副词条不能省略或替换为任意词条。
- 无法证明自然生成合法性的条目不进入构造队列；等级限制只公开可证明的自然范围。
- `7net / Seven Net` 由 2.0.2 中/英/日本地化表、`dlc.tbl` 与真实存档共同确认为 Seven Net Shopping 零售商特典，不是占位数据。真实记录使用特殊 `flags=22`，普通构造器只写 `flags=2`，因此保留真实显示但禁止伪造构造。
- `0x40223C28` 按本地 `ids.txt` 和生成表修正为 `SKILL_143_00`，不再误标为 `SKILL_166_50`。
- 因子/祝福离线生成器有实例级串行化、队列副本返回、后端数量上限、写后回读和失败恢复；失败重试不重复追加。

### 5. 运行时安全和监测边界

- 因子、祝福、召唤石、上限突破、货币、药水、素材消耗、巴武掉落和怪物增强写入统一加入进程租约/所有权校验。
- 写入前后复核 PID、进程创建时间、模块基址、捕获地址和旧值；页面令牌不能跨进程复用。
- 同一进程上的互斥运行时功能进入冲突门禁；关键写入进行回读，失败时回滚，无法证明回滚成功时隔离当前进程实例。
- 代码洞安装使用“从未发布 / 已安装 / 发布尝试后已恢复 / 需恢复”状态。只有能证明从未发布的代码洞才立即释放；可能被任意线程看到过的入口即使已恢复，也把代码洞退役保留到进程断开，避免执行中线程 UAF。
- 队伍监测仅使用只读路径，不安装 Hook。
- 选中素材/关键物品监测会临时安装两个只用于捕获选中地址的 Hook，并在停止/断开时恢复入口。它不修改数量、标志或存档，但不得被描述为“完全零内存写入”。

### 6. UI、自适应与立绘

- 保留羊皮纸和场景背景，继续使用纸张、金色描边、按钮、输入框、卡片和弹窗的统一设计令牌。
- 正常工具页保持“左侧功能面板 + 右侧大幅立绘”双栏结构；立绘不放进小盒子，配装编辑时才隐藏。
- 标题栏右上保持最小化、最大化/还原、一键全屏、关闭四个窗口控制；窄窗口中首页入口和 Q 版形象仍应可达。
- 祝福石内存编辑的“启用读取后选中目标”使用紧凑行内提示，不再以居中大空框占据主内容。
- 新增的 CT 战斗、角色、任务和监测页按功能分别映射立绘与表情包，生成依据 `D:\gbf\GBFR-Relink-Portrait-Skill-Portable-20260718-v3.zip` 的分阶段工作流。最终资产哈希去重、透明边界和真实 Wails 截图复核仍列为发布 TODO。
- 外部模型的版式意见只作为设计输入；最终是否通过仍以真实 Wails WebView2 的尺寸矩阵、控件可达性和无裁切/重叠证据为准。
