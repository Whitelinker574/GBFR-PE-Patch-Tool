本项目是面向《碧蓝幻想：Relink》的非官方桌面工具，数据结构与偏移基于 DLC 2.0.2 版本整理。三种工作方式互相独立、边界固定：离线存档编辑要求完全退出游戏后进行，写入经过备份、校验与原子替换；实时写入需要连接正在运行的游戏进程，直接改动内存；只读监测只读取运行时数值，用于观察和公式校准，不产生任何写入。

工具只操作本地存档和游戏进程中已有的数据，不会解锁账号尚未拥有或官方尚未开放的 DLC 内容。功能在 DLC 2.0.2 环境下开发并做过有限自测，未覆盖全部实机场景；写入前请备份存档与目标文件，写入后用页面回读核对结果。

## 快速开始

1. 从 [Releases](https://github.com/Whitelinker574/GBFR-PE-Patch-Tool/releases/latest) 下载 Windows amd64 压缩包，并核对随包提供的 SHA-256。
2. 修改存档前完全退出游戏并备份存档；修改 EXE 前备份目标文件；实时功能仅在本地单人环境使用。
3. 打开程序后按用途选择左侧入口：`存档修改（离线）`、`内存注入（实时）` 或 `内存监测（只读）`。
4. 每次写入前确认存档位、角色、物品和槽位；写入后使用页面回读结果确认目标值。

默认存档目录：

```text
C:\Users\<用户名>\AppData\Local\GBFR\Saved\SaveGames\
```

程序会同时显示识别出的多个存档位，也可以手动浏览文件。离线写入使用备份、校验和修复、临时文件、原子替换和重新读取；不要在游戏运行时同时修改同一份存档。

### 存档与配装

![配装预设与三存档入口](docs/screenshots/loadout-presets.png)

- 因子、祝福和召唤石在存档、内存与配装构造入口共用同一套 2.0.2 目录。
- 配装工作台可读取每个角色的 15 个预设槽，并编辑武器、12 个因子、4 个技能、三方向专精与召唤石。
- 单套配装可以导入或导出；导入只匹配当前存档已经拥有的资源，不做存档批量搬运。
- 天然组合、常见等级与召唤石合法性只作提示。只要值可以编码，用户选择默认允许写入；目标所有权、容量、整数范围、校验和与写后回读仍是硬性检查。
- 未进入 DLC 的存档可能存在预分配记录，但写入记录不等于解锁系统。游戏是否接受和展示该记录仍由游戏自身状态决定。

配装估算会把存档可证明的角色成长、武器、因子、专精、上限突破和召唤石来源分项展示；能够从游戏读取最终 HP、攻击力、暴击率和昏厥值时，会与草稿并排对照。没有现场证据的结果继续标为估算、候选或未闭环，不冒充真实最终公式。

### 实时编辑与补丁

![战斗规则补丁](docs/screenshots/patch-combat.png)

- 所有实时写入绑定当前进程、连接所有权令牌和选中目标，切换角色、重载存档或目标失效时拒绝沿用旧快照。
- 战斗规则、角色机制、任务便利三个补丁页共用一条常驻连接；切换页面不会自动恢复已开启功能。只有明确“恢复全部并断开”或退出程序时才统一清理。
- 补丁目录锁定 DLC 2.0.2 的原字节、特征码和写后回读。冲突功能由目录显式管理，不能为通过测试而跳过版本守卫。
- 自动完美格挡的连招中断修复、单人队伍监测和训练场防御样本已经完成现场重复验证；未逐项实机覆盖的运行时功能不会被笼统标成全场景通过。

实时功能仅用于本地单人环境。重启游戏后需要重新连接；不要把运行时修改带入联机房间或影响其他玩家。

### 只读监测与公式采样

![角色公式采样](docs/screenshots/formula-sampler.png)

公式采样器只申请进程查询与读取权限，不安装 Hook、不注入、不写内存，也不修改存档。它可以连续读取最终 HP、攻击力、暴击率和昏厥值，记录一次稳定前值、过渡期和稳定后值，并保留严格 A1/B1/A2/B2 模式用于可重复验证。导出的证据会删除 PID、模块基址、绝对地址、用户名和本地路径。

防御、减伤、伤害上限和条件技能不一定改变人物面板，因此需要训练场受击或木桩样本。四项面板没有变化只算负观察，不能据此断言效果不存在。

### 游戏文件维护

游戏文件维护页用于识别目标 EXE、显示版本边界、创建 `.bak`、应用已审核补丁和恢复备份。遇到未知 EXE、特征不唯一、原字节不符或地址越界时会停止，而不是尝试写入相似位置。Steam 文件验证也可用于恢复原始文件。

## 写入与证据边界

| 层级 | 工具会做什么 | 不会承诺什么 |
| --- | --- | --- |
| 存档写入 | 备份、事务写入、校验和、原子替换、回读 | 不代替游戏解锁 DLC 系统，不保证任意非法组合被游戏消费 |
| 运行时写入 | 版本守卫、所有权令牌、目标快照、写后回读、恢复原值 | 不支持联机使用，不把未知版本当兼容版本 |
| 只读监测 | 查询进程并读取有限对象、稳定性验证、脱敏导出 | 不扫描或导出整个进程，不把候选字段当最终公式 |
| 配装估算 | 展示来源、证据等级和与实读差值 | 不用固定截图值或未经验证的缩放硬凑一致 |

详细边界见 [公式与证据等级](docs/FORMULAS_2.0.2.md)、[公式采样操作说明](docs/角色公式采样操作说明.md) 和 [存档/内存目录一致性](docs/evidence/save-memory-table-parity.md)。

## 项目结构

```text
.
├─ main.go                            仅负责嵌入前端并启动 Wails 后端
├─ internal/backend/
│  ├─ README.md                       后端功能域与文件前缀索引
│  ├─ loadout*.go / save_*.go         配装、存档解析和事务写入
│  ├─ sigil_*.go / wrightstone_*.go   因子与祝福的存档/内存通道
│  ├─ summon_*.go                     召唤石目录、存档和内存通道
│  ├─ runtime_*.go / formula_*.go     运行时补丁、监测和公式证据
│  ├─ *_test.go                       与对应功能同包的自动化回归测试
│  ├─ data/                            嵌入的 2.0.2 目录、布局和证据
│  └─ resources/                       随程序嵌入的原生资源
├─ frontend/
│  ├─ src/components/                Vue 页面与共享组件
│  ├─ src/*.test.js                  前端目录、交互和安全回归测试
│  └─ wailsjs/                       Wails 构建时生成的绑定（不入库）
├─ src_dll/                           可复现的 patch_core 原生组件源码
├─ tools/                             数据审计、目录生成与图标同步脚本
├─ docs/                              维护文档、公开截图与脱敏证据
├─ THIRD_PARTY_NOTICES.md              第三方组件、原生依赖与来源边界
├─ build/                             Wails 元数据、图标与本地构建输出
└─ .github/workflows/ci.yml           Go、前端和静态检查
```

`tools/` 只包含可复现的数据与发布维护脚本；临时文件应保存在 Git 忽略的本地目录。`*_test.go` 和 `*.test.js` 是随功能代码维护、但不会编入成品的自动化验证，不属于用户运行脚本。完整说明见 [架构文档](docs/ARCHITECTURE.md) 和 [后端文件索引](internal/backend/README.md)。

## 本地构建与验证

环境：Windows amd64、Go 1.25+、Node.js/npm、Wails CLI v2.13、WebView2 Runtime。只有重建 `src_dll/patch_core` 时需要 Visual Studio/MSBuild。

```powershell
cd frontend
npm ci
npm run build
cd ..

go test ./...
go vet ./...
node --test frontend/src/*.test.js
wails build -platform windows/amd64 -clean
```

产物位于 `build\bin\GBFR PE Patch Tool.exe`。发布前还应启动成品程序，核对版本号、主要页面、更新入口和 SHA-256。
