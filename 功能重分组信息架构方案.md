# 功能重分组信息架构（内存注入 vs 存档修改）

> 由 6-agent 工作流产出。顶层按 存档修改/内存注入/工具 三分，合并重复功能，剔除遗留注释。

## 顶层分组
顶层三分（对应 componentLoaders/toolMeta.group/HomeJournal 三处导航需同步改）：

【内存注入 · 运行时（需连接游戏）】前置=process-attach（CharaAttach/CharaDetach，app.go:976/1219，被 MiscTools/SummonEditor 复用），是所有运行时功能的连接闸门，应做成整个顶层组的统一「连接游戏」入口而非每功能各写一遍。定位=改的是进程内存/走游戏保存流程，退游戏即失效。台前只放已适配的稳定项，未适配项折叠进本顶层内的「实验/诊断」子组（用状态标签，不再单列 nav）。

【存档修改 · 离线（改存档文件）】定位=退游戏后离线改文件，可批量、可回滚。所有存档功能共用同一外壳：选存档→LoadSave 解析（save_app.go:147，被 save_app/loadout/loadout_write/progression_editor/sigil_gen/wrightstone_gen 复用）→原地改值→autoSnapshotBeforeSaveWrite 自动快照→FixChecksums→写回→回读验证。这是良性复用，正好支撑统一分组。社区最高频养成需求集中在此，应占首屏头牌。

【工具 · 元信息/基础设施】定位=不直接改游戏数值：存档快照安全网、版本适配诊断、游戏 EXE 维护、语言设置。EXE 备份/二进制补丁与存档备份是两条不同对象的路径，必须在文案上分清（EXE vs 存档），并把已弃用的二进制写入下沉到诊断区。

首页（HomeJournal.vue，PatchTool.vue:527）应改为「两条路径导引」：离线存档修改 / 实时内存注入，与本三分顶层一致；当前 HomeJournal 手写 chapters+small-tabs 是第三份独立导航（HomeJournal.vue:7-12,37-39），重构时必须与 nav、toolMeta.group 同步，否则台前入口与侧栏分组打架。

## 分组明细

### 运行时·因子（内存）（内存注入）
- 功能：sigilMemory（因子即时编辑，SigilMemoryGenerator.vue）+ loadout（因子配装录制/复刻，SigilLoadoutRestore.vue）合并为同一面板两模式
- 理由：两者后端完全同源，都只 import sigil_memory.go 的 SigilMemoryEnable/GetOptions/GetStatus/Update（SigilMemoryGenerator.vue:2-8 与 SigilLoadoutRestore.vue:3-8）。sigilMemory=单枚当前选中因子实时改；loadout=同引擎上加录制12枚/导出JSON/逐枚复刻的批处理层。当前拆成两个并列 tab 是重复入口，应合成一页：模式A单枚精修 / 模式B 12格配装导入导出。命名务必与存档侧『因子修改』区分为『即时编辑/运行时』。核心，放台前。

### 运行时·召唤石（内存注入）
- 功能：summon（SummonEditor.vue → summon_memory.go SummonGetOptions/GetAll/Update）
- 理由：读游戏内召唤石背包改因子/副参数/等级，走游戏保存流程。纯运行时无存档侧（存档召唤石库存 1456-1460 仅 MEMORY 记存疑、代码未实现），勿虚设存档分组。核心。

### 运行时·角色上限突破（内存注入）
- 功能：overlimit（OverLimit.vue → overlimit.go Scan/Enable/SetSlot/Commit）
- 理由：读突破界面4能力槽，callRemoteOneArg 按游戏原流程保存。经核实纯运行时，progression 内无任何 overlimit/limitBreak 存档实现——用户假设的『存档侧上限突破』不存在。注意与 progression 的『武器超凡』(weaponTranscendence, progression_editor.go:38/199-203) 命名区分：一个是角色突破(运行时)、一个是武器超凡(存档)，对象不同勿合并。核心。

### 运行时·实时资源（内存注入）

## 迁移·新 nav 结构
顶层保持现有 2 级模型（sidebar 组 → tool-switcher items），把当前 4 组重切成「存档 / 内存 / 工具」三组，直接替换 PatchTool.vue:220-225 的 navigation：

const navigation = computed(() => [
  { id:'save',   mark:'档', label:'存档修改（离线）', caption:'退出游戏后改存档文件',
    items:['progression','sigil','wrightstone','loadoutPresets','chara','save'] },
  { id:'memory', mark:'注', label:'内存注入（实时）', caption:'连接游戏改进程内存',
    items:['sigilMemory','loadout','summon','overlimit','runtime'] },
  { id:'tools',  mark:'具', label:'工具与设置', caption:'版本诊断 · EXE维护 · 语言',
    items:['compatibility','patch','language'] },

要点/依据：

 - i18n 键删除有连带风险：i18n.js:1489/1510、i18n-ui.js 多行——删前必须全局搜索键名确认零引用，删掉在用键会让界面显示原始 key 或空串。
